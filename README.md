# Jellyfin Ultimate Parental Control

A powerful Parental Control plugin for Jellyfin (10.11.6+) that goes far beyond native rating restrictions. It acts as an automated content tagger that intercepts items based on Keywords, Genres, Studios, or strict Whitelists, allowing you to easily block those tags per-user using Jellyfin's built-in parental controls.

## Features

- **Real-Time Tagging**: Monitors library additions and updates as they happen, instantly tagging inappropriate content before anyone can watch it.
- **Keyword Deep Scanning**: Scans titles and overviews/descriptions for forbidden keywords (e.g., "gore", "violence") and automatically slaps a restricted tag on them.
- **Genre & Studio Blacklisting**: Automatically tag items generated by specific studios (e.g., adult networks) or matching certain genres.
- **Absolute Whitelist Mode**: The "Nuke" option. If an item doesn't contain explicitly safe genres, studios, or keywords, it gets hit with a fallback restriction tag. Perfect for ensuring specific accounts only see pre-approved content.
- **Fully Integrated UI**: Clean, built-in configuration page directly inside Jellyfin's Dashboard.
- **Background Sync Task**: A dedicated scheduled task that can retroactively scan your entire massive library safely without hitting database bottlenecks.

## How It Works

This plugin operates on a highly stable tag-synchronization pattern:
1. You set up your Blacklists/Whitelists in the plugin settings.
2. The plugin automatically applies specific "Tags" (e.g., `RESTRICTED_WORD` or `REQUIRES_APPROVAL`) to media matching your rules.
3. In the native Jellyfin Server User settings, you go to your intended user's **Parental Controls** and simply block those exact tags.

## Manual Build

Prerequisites: `.NET 9.0 SDK`

If you just want to build the code locally:
```powershell
dotnet restore .\jellyfin-genres-removal.sln --configfile .\NuGet.config
dotnet build .\jellyfin-genres-removal.sln -c Release --no-restore
```

## Release & Publish (For Remote Repository Setup)

We provide an automated script to build your plugin, zip it, and securely inject the exact MD5 checksum directly into your `manifest.json`.

1. Generate the exact version payload and manifest:
```powershell
.\pack-plugin.ps1 -Version "0.4.0.0" -DownloadUrl "https://github.com/az2oo1/jellyfin-genres-removal/releases/download/v0.4.0.0/JellyfinGenreRestriction_0.4.0.0.zip" -UpdateManifest
```

2. Go to your GitHub Repository -> **Releases** -> Create a new release (e.g., `v0.4.0.0`).
3. Upload the freshly generated `artifacts/JellyfinGenreRestriction_0.4.0.0.zip` directly to the GitHub release.
4. Commit and push the updated `dist/manifest.json` to your repository's `main` branch.

**(Important: Do not run `dotnet build` or the script again after this, as the zip file's timestamp and MD5 checksum will change and break the install).**

## Installation in Jellyfin

1. In your Jellyfin Dashboard, go to **Plugins** -> **Repositories** -> Add (`+`).
2. Enter the repository details:
```text
Name: Ultimate Parental Control
Repository URL: https://raw.githubusercontent.com/az2oo1/jellyfin-genres-removal/main/dist/manifest.json
```
3. Go to the **Catalog** tab, find the plugin, and install it.
4. Restart Jellyfin.
